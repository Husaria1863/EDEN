# Contributing to EDEN

EDEN is built through a fork-and-pull-request workflow. Contributors should make changes in their own fork and submit a pull request for review.

## Workflow

1. Fork the repository.
2. Clone your fork.
3. Create a new branch for your change.
4. Make your changes.
5. Test the project in Godot.
6. Commit and push your branch.
7. Open a pull request to the official EDEN repository.

## Contribution Types

Accepted contribution types include:

- Bug fixes
- Gameplay features
- 3D assets
- Animations
- Environment objects
- Camera/player improvements
- Documentation
- Research/canon proposals

## Rules

- Do not commit broken scenes.
- Do not commit exported builds.
- Do not add assets you do not own or have permission to use.
- Any object the player can touch must include collision.
- Keep features consistent with the project direction.
- Test the main scene before submitting a pull request.

## Current Project Direction

EDEN is currently focused on building a simple early-world simulation foundation. Features should support movement, environment building, survival systems, early-human interaction, or world simulation.
